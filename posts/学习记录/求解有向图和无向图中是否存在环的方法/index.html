求解有向图和无向图中是否存在环的方法

2021-12-02
3 minute read

求解无向图中是否存在环

无向图中是否存在环，可以通过DFS来实现。

由于他是无向图，所以每次开始一次新的递归即进入一个新的连通子图，在该次dfs中可以通过判断新纳入的结点是否与所在连通子图中其他已经访问过的结点存在边，如果存在则证明存在环。

 1int mat[maxn][maxn], vis[maxn];
 2int dfs(int v) {
 3    vis[v] = 1;
 4    for (int i = 0; i < num; i++) if (mat[v][i]) {
 5        if (vis[i] || dfs(i)) return 1; // 该连通子图新纳入的结点与原连通子图中的某一结点有边，即产生了环
 6    }
 7    return 0;
 8}
 9
10memset(vis, 0, sizeof(vis));
11int flag = 0; // 无环标志
12for (int j = 0; j < num; j++) {
13    if (!vis[j]) flag = dfs(j); 
14    // 因为是无向图，所以每开始一次新的递归即进入一个新的连通子图，一定不会访问到其他连通子图的结点
15    if (flag) break; // 如果在该连通子图中找到环则直接结束递归
16}

求解有向图中是否存在环

这里我也尝试过使用如上DFS的解法，但似乎行不通。

因为有向图的DFS，每开始一次新的递归未必进入一个新的连通子图！

基于这一点，使用上述无向图的方法是行不通的，因为如果以后的递归中访问一个结点时，该结点可以指向其他连通子图访问过的结点，导致误判为出现了环，而实际上并没有。

所以需要转换思路。

转念一想，不是还有一个拓扑序的思想吗？！

对一个有向图来说，如果存在拓扑序则该图一定不存在环。

所以完全可以使用拓扑排序的算法来判断环的存在与否。

由于存在拓扑序的一个充要条件是：

算法进栈结点数等于总结点数

所以比较进栈的结点数和总的结点数即可！

 1int topo() {
 2    stack<int> st;
 3    int ncount = 0;
 4    for (int i = 0; i < num; i++) if (!in_num[i]) st.push(i); // 找到第一个入度为0的结点
 5    while(!st.empty()) {
 6        int k = st.top(); st.pop(); ncount++;
 7        for (int i = 0; i < num; i++) {
 8            if (mat[k][i]) 
 9                if (--in_num[i] == 0) st.push(i); // 如果减去1后入度为0则属于拓扑序结点，进栈
10        }
11    }
12    return ncount;
13}
