关于数论的一些简单算法

2020-11-17
6 minute read
Algorithm

关于素数

1.埃氏筛法

问1000000000000以内有多少个素数？

运用朴素算法必TLE，这时考虑埃氏筛法。

算法思路：

1.建立is_prime[]数组，初始化为true；

2.从2开始筛取，(注意从2开始很重要，因为2为素数，否则需要改变相应后续操作)，若为true，则继续判断是否为素数，若为素数，则将所有该素数的倍数置为false。

2.区间筛法

问[21479862, 21499877)之间有多少个素数？

这时若采用埃氏筛法，会浪费大量时间计算前面未涉及的区间，这时考虑区间筛法。

算法思路：

1.建立is_prime_small[]和is_prime[]数组，初始化为true；

2.对区间[a, b)，只需要对 [2, 根号b) 在is_prime_small[]进行埃氏筛法，同一循环内对[a, b)中相应素数倍数元素置为false。

相应代码：

 1typedef long long ll;
 2bool is_prime[MAX1];
 3bool is_prime_small[MAX2];
 4void sieve(ll a, ll b) {
 5	for(int i = 0; (ll)i * i < b; i++) is_prime_small[i] = true;
 6	for(int i = 0; (ll)i * i < b; i++) is_prime[i] = true;
 7	for(int i = 2; (ll)i * i < b; i++) {
 8		if(is_prime_small[i]){
 9			for(int j = 2 * i; (ll)j * j < b; j += i) is_prime_small[j] = false;
10			for(ll j = max( 2ll, (a + i - 1) / i) ) * i; j < b; j += i) is_prime[j - a] = false;
11		}
12	}
13}

注意点：

1.max( 2ll, (a + i - 1) / i ) * i 其中的2ll为隐式地将max的参数类型转换为long long 型

2.（a + i - 1) / i * i 可以得到的是a或a以后的第一个素数的倍数：

当a为该素数的倍数时：

设倍为x，则a等于ix

(ix + i - 1) / i * i

即为 [ix / i + (i - 1) / i] * i = ix

反之：

设a等于ix + t

则有t >= 1 && t < x

(ix + t + i - 1) / i * i

即为 [ix / i + (i - 1 + t) / i] * i = （x + 1) * i = ix + i > ix + t

即ix + i 为 ix + t后第一个i的倍数

关于模

取模的运算

（a+b)%p = (a%p+b%p)%p +可以换为-和*

当处理的数据非常大时，可以一步一步取模，在较小数据下就可以得到最终结果

快速幂运算

问n是否满足x^n mod n = x (1 < x < n)？

3^10 – 9^5 – 9 * 9^4 – 9 * 81^2 – 9 * 6561^1 – 6561^0

result = 9 * 6561

可以推出幂的结果等于变化中所有当指数为奇数时底数之积

相应代码：

 1typedef long long ll;
 2ll mod_pow(ll x, ll n, ll mod) {
 3	ll res = 1;
 4	while(n > 0) {
 5		if(n & 1 == 1) res = res * x % mod;
 6		x = x * x % mod;
 7		n >>= 1;
 8	}	
 9	return res;
10}

注意，运用位运算可以提高效率！

关于最大公约数

gcd辗转相除法求最大公约数

思路

反复交换取余，直到小的数为0.

1int gcd(int a, int b){
2	if(b == 0) return a;
3	return gcd(b, a % b);
4}

exgcd扩展欧几里得算法

先介绍贝祖定理：

若a,b为整数，则一定存在整数x,y使得ax + by == gcd(a,b)。

即若ax + by == m有解，则m一定为gcd(a,b)的若干倍。

有a, -a, b, -b四个整数，各用几次可以使得ax + by == 1?

有上述思想则可知gcd(a,b)一定等于1，可编写一个返回值为gcd（a,b）同时递归计算x和y的函数。

关于求出x和y推导过程：

由ax + by == gcd(a,b) (1)

得bx1 + (a % b) y1 == gcd(a,b)

由a % b == a - (a / b) * b带入得

ay1 + b * (x1 -(a / b) * y1) == gcd(a,b) (2)

由（1）（2）可得：

x == y1

y == (x1 - (a / b) * y1)

由等式可知上层的x,y可以由下层的x1,y1来求,则可以通过递归来实现。

又由（1），当b == 0, a * 1 + b * 0 == a == gcd(a,b)

则此时x == 1, y == 0, 一定会出现在递归末尾。

 1int exgcd(int a, int b, int &x, int &y){
 2	int d = a;
 3	if(b != 0){
 4		d = exgcd(b, a%b, x, y);
 5		int temp = y;
 6		y = x - (a / b) * y; //上一层的y
 7		x = temp; //上一层的x
 8	}else{
 9		x = 1; y = 0;
10	}
11	return d;
12}
