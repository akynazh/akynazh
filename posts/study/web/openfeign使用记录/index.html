为什么要使用OpenFeign

之前在消费端使用RestTemplate时，每次请求都要进行诸如restTemplate.postForObject(PAYMENT_URL + "/payment/create", payment, CommonResult.class);这样的调用，需要指定较多参数，当一个接口调用中需要非常多这样的请求时，会比较繁琐，而且这种方式不够抽象。

OpenFegin利用面向接口编程的思想，抽象化，简化了上述操作。

  3. 编写服务接口

    1. 首先在启动类开启@EnableFeignClients注解；
    2. 编写服务接口：
      • 添加@FeignClient注解，值为对应微服务名；
      • 方法对应微服务Controller下的方法即可。

    @Component
@FeignClient(value = "CLOUD-PAYMENT-SERVICE")
public interface PaymentService {
    @GetMapping("/payment/get/{id}")
    CommonResult<Payment> getPaymentById(@PathVariable("id") Long id);

    @PostMapping("/payment/create")
    CommonResult<Integer> createPayment(@RequestBody Payment payment);
}

  4. 调用接口

    注入PaymentController接口，然后即可调用它的方法。

    @RestController
@Slf4j
public class PaymentController {
    @Resource
    private PaymentService paymentService;

    @PostMapping("/consumer/payment/create")
    public CommonResult<Integer> create(@RequestBody Payment payment) {
        return paymentService.createPayment(payment);
    }

    @GetMapping("/consumer/payment/get/{id}")
    public  CommonResult<Payment> getPayment(@PathVariable("id") Long id) {
        return paymentService.getPaymentById(id);
    }
}
