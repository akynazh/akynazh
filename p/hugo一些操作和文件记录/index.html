hugo目录大致功能

-blog/

- archetypes/: 标准类型目录，生成对应文章页面时会由这个目录里的md模板生成

- assets/: 全局物件目录，存放全局图像（不包含image加载的，主要是图标等），scss样式，ts脚本的等的目录，可以从theme对应的assets里的文件中复制同样的文件目录进行补充添加覆盖等

- content/: 文章和页面目录

- data/: 数据模板目录，需要加载数据库时有用

- layouts/: 布局模板目录，可以从theme对应的layouts里的文件中复制同样的文件并加以覆盖升级

- public/: 部署网站内容到github的目录

- resources/: 缓冲目录，加速hugo命令完成

- static/: 加载图像文件时对应的根目录，如：image: /image/1.jpg  ->  static/image/1.jpg

- themes/: 存放各种主题

- config.yaml: 全局配置文件

hugo一些操作记录

- hugo new \post\*.md 生成文章or页面

- hugo 编译部署到public

- hugo server 编译并开启本地服务器预览
