SpringBoot中关于controller方法返回值的辨析

阅读时长: 2 分钟

返回值为重定向

1

return "redirect:/success"

重定向到/success请求。也就是说还要执行/success请求对应的方法。

一般表单提交成功，可以使用重定向请求，避免用户多次发起请求。

而如果表单提交失败，一般可以使用请求转发到原来的页面，也可以使用下面返回视图对象的方法。

但是，值得注意的是，只是返回视图对象的话，必须注意一些问题：

如下请求路径：

/theater/watch?videoId=1

如果请求失败：

  • 可以进行重定向到/theater。
  • 而如果不进行重定向，只是返回视图对象：
    • 如果原来是get请求，用户按F5刷新页面则会重复访问请求，所以必须导向正确的URL。
    • 如果原来是post请求，且url路径不改变则可以这样，如果url路径不变用户刷新将导致404错误。

因此，返回视图对象必须注意一些路径上的细节。

返回视图对象

1

return "success"

注意必须已经启动了thymeleaf引擎。

该方法将转发到 classpath:/templates/success.html页面。

值得注意的是，如果加载success页面含有需要解析的modelAttribute中的对象，则该方法中必须往modelAttribute添加该对象，否则会报错。

所以如果多个页面都可能访问到这个也页面，则最好别使用modelAttribute注入对象，如果这样必须先注入视图对象所需要的model对象。

否则一律需要redirect到有注入该对象的方法中去才能正确解析页面。

另外，可以通过将model对象放到session中，同时视图对象中通过session值来获取对象，即可。
