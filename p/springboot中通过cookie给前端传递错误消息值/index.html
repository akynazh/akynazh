SpringBoot中通过cookie给前端传递错误消息值

Dec 31, 2021
阅读时长: 2 分钟

Author: akynazh

个人网站: akynazh

后端操作方法

这里我们需要后端在接收一些请求并做出判断后，向前端反馈请求情况。

考虑到安全性，一般是不允许前端访问session值的，所以我们通过cookie给前端反馈：

如下是一个controller方法中的片段：

1
2
3

response.addCookie(new Cookie("errorMsg", "邮箱已经被注册"));
request.getRequestDispatcher("/success").forward(request, response);
return null;

我们给前端添加了一个cookie值。

这里必须注意的是，不能在添加完cookie后进行重定向，否则无法正确成功传递cookie值。必须使用请求转发的方式，这也刚好和表单请求失败后的处理情况相合。

前端操作方法

下面我们通过javascript方法进行获取并发出提示并删除cookie：

注意，执行完提示后必须删除该cookie值，否则以后访问页面将一直提示。

1
2
3
4
5
6
7
8

$(document).ready(function () {
    let errorMsg = $.cookie('errorMsg');
    if (errorMsg != null) {
        let myPath = window.location.host + '/';
        $('.alert').html(errorMsg).addClass('alert-info').show().delay(1500).fadeOut();
        $.cookie('errorMsg', '', {path: myPath, expires: -1});
    }
})

这里很关键的是需要指定path值，不同路径下生成的是不同的cookie值，即使它们同名为errorMsg。

所以，为了删除同为errorMsg名的cookie值，path必须指定为所有路径。

这里有个坑， ‘/’ 似乎不能指定为所有路径, 而使用完整路径则可以。

关于获取完整路径的方法：

1

let myPath = window.location.host + '/';

window.location.host:

  • ‘https://akynazh.site/p/linuxTalk' -> ‘akynazh.site’
  • ‘http://localhost:8080/index’ -> ‘localhost:8080’
